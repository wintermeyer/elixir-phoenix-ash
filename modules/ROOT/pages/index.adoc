= Preface
Stefan Wintermeyer <sw@wintermeyer-consulting.de>

Learning Phoenix is a steep climb. No sugar-coating. This book
is my attempt to make that ascent less steep — a beginner's
guide to Phoenix, written in the same step-by-step shape I use
when I run Phoenix trainings. A bonus track on
link:https://hexdocs.pm/ash/[Ash] is included, because in
practice the two ship together often enough that it pays to
see both.

* *Step-by-step.* No prior Phoenix knowledge required.
* *Bite-sized "Hello World!" code examples.* Crafted for
  comprehension just by reading the code, with notes added for
  clarity. No abstract programming theory.
* *Simplicity is key.* My goal is to get you productive ASAP.
* *Not a beauty contest.* We're not investing much time in
  aesthetics. This is not a CSS design book.

**Elixir before Phoenix.** Phoenix is built on Elixir, and this
book assumes you know Elixir basics. If pattern matching, the
pipe operator, GenServer or Mix are still strange, start with
link:https://wintermeyer-consulting.de/elixir/book/[Elixir] —
the language primer written as the prerequisite for this book
— and come back here for the framework.

Coming from Ruby on Rails? Phoenix is conceptually the Rails
of the BEAM. The Rails-specific sibling book is at
link:https://wintermeyer-consulting.de/rails/book/[Learn Ruby
on Rails].

The sidebar lets you jump to any chapter at any time — nothing
is locked, later chapters don't assume you memorised the
earlier ones. That said, the book is written to be read front
to back: each chapter sets up the next. If a section doesn't
resonate, skip it — you can circle back later if needed.

== Don't be a stranger

A book like this gets better every time a reader sends me a note.

* Did a chapter click, or did it leave you more confused than
  before? Either answer is useful.
* Found a typo, a broken example, code that no longer works on
  the current Phoenix or Ash version, a link that 404s?
* Missing a topic you expected? Or curious about something this
  book doesn't cover?
